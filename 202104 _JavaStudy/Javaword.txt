■ 開発

ウォーターフォール開発　
　システムの開発工程を「計画」「外部の設計」「内部の設計」「プログラム自体の設計」「プログラミング」
　「テスト」などの行程に分けて順番に進行する方法・

アジャイル開発
　現在よく使われているソフトウェアやシステム開発手法の一つ。機能単位の小さなサイクルで、計画から設計・
　・開発・テストまでの工程を繰り返す

マイグレーション
　新しいシステムに切り替えたり、システムやデータを別のプラットフォームに移行したりする意味

デグレーション
　プログラムを変更したら、前よりおかしくなる状態。
　
デプロイ
　ソフトウェア開発の工程のうち、開発した機能やサービスを利用できる状態にする作業。

コンフリクト
　競合、衝突。複数の装置やプログラミングなどが同じ資源を同時に利用して競合状態になってしまう状態。

「環境」
　何かをするための場

・開発環境（ローカル）
　機器やソフトウェア、システムを開発するために必要な機器やソフトウェアなどの組み合わせ。開発者の
　作業環境。

・検証環境（ステージング）
　情報システムやソフトウェアの開発の最終段階で検証用に用意される、実際の運用環境と変わらない環境。

・本番環境
　実際にシステムが稼働している環境のこと。

「設計書／仕様書」
　システム開発をしていくに当って重要になる計画を立てる工程。

・要件定義
　顧客の要求を把握してシステム要件を確定すること。

・基本設計書
　ユーザとの使用確認を目的に、業務の整理やデータ、画面レイアウトなどの枠組みを設計する作業。

・詳細設計書
　基本設計書で作成したドキュメントはそのまま利用され、さらに詳細機能を表すためのドキュメントを追加する
　作業。

「テスト」
　システムの不具合をなくすために行われる動作確認。

・単体テスト（UT）
　関数やメソッドを単位とし、一つ一つの処理が正しく機能しているかを確認する。

・結合テスト（CT／IT）
　いくつかのモジュールを組み合わせて、意図した結果が帰ってくるかを確認する。

・総合テスト（ST）
　開発したシステム全体を通して意図した通りに動作するか、仕様書通りの機能を実現できているか、
　性能要件を満たしてるかどうかを確認する。

エビデンス／証跡
　証拠・根拠。テスト工程で、システムが正しく動作している。または不具合が生じたことを示す証拠となる
　書類やデータ。

データパッチ
　プログラムやデータを更新・修正するために新しい版との相違点を抜き出したデータのこと

Access
　機器やソフトウェアなど利用可能な状態にすることをいう。

マクロ
　関連する複数の操作や手順、命令などを一つまとめ、必要に応じて呼び出すことができるようにする機能。

VBA
　マクロを実現させるための機能。

GAS（Google Apps Script）
　Googleが提供しているプログラミング環境。

「API」
　ソフトウェアからOSの機能を利用するための仕様またはインターフェイスの総称。

・WEB API
　コンピュータプログラムの提供する機能を外部の別のプログラムから呼び出して利用するための手順・APIの一つ。
　HTTPなどWebの技術を用いて構築されたもの。

・REST API
　RESと呼ばれる設計原則に従って策定されたもの。

・SOAP
　異なるコンピュータ上で動作するプログラム同士がネットワークを通じてメッセージを伝え合い、連携して動作するための
　通信プロトコルの一つ。

「Ajax」
　あるWebページを表示した状態のまま、別のページや再読み込みなどを伴わず、Webサーバ側と通信を行い、動的に表示内容を
　変更する手法。

・XML
　文章の見た目や構造を記述する為のマークアップ言語。

・JSON
　コンピュータ間でデータのやりとりをする時に使うフォーマット。

・ステータスコード
　システムが処理結果や現在の状態を外部に知らせるために発する数字や短い符号。

・クロスオリジン（Cross-Origin）／CORS（Cross-Origin Resource Sharing）
　自身のオリジンから見た、別のオリジンからのリクエストを許可するか決定する仕組み。
　オリジンとは、スキーム、ホスト、ポート番号の組み合わせでドメインとは少し異なる。

バリデーション
　対象がその仕様や文法などに照らして適切に記述・構築されているかどうかを検証すること

「データ連携」
　異なるアプリケーションやシステムをまたいでデータを共有、活用するために、それぞれの言葉や解釈の
　垣根を取り払うプロセス。

・Daemon
　UNIX系OSで常駐して特定の機能を提供するプログラム。
　利用者の操作とは無関係に処理を行う。

・CRON
　DAEMONの一種。利用者の設定したスケジュールに従って指定されたプログラムを定期的に起動するもの

・バッチ
　一回分にまとめるという意味。バッチ処理はあらかじめ登録した一連の処理を自動的に実行する処理方式。

・ログ
　コンピュータの利用状況やデータ通信などの情報の記録をとること。

・CSV
　カンマで区切って書いたテキストファイル・データのこと。

・TSV
　タブで区切って書いたテキストファイル・データのこと。

プルリクエスト（マージリクエスト）
　分散バージョン管理システムの機能の一つ。コードなどを追加・修正した際、本体への反映を他の開発者に依頼する機能・

■ チケット管理ツール
 タスク管理ツールのこと。プロジェクト毎に発生する各種タスクを発行して、製作者に対して仕事を依頼するためのもの。

・asana
　イベントやキャンペーンに携わるチームメンバーの日々の仕事をサポートするためのワークマネージメントツール

・Backlog
　実施するべきとされながら、未処理、未着手のまま放置されている作業や業務、案件などのこと。

・Jooto
　タスク毎にカードを作成し管理していくカンバン方式のツール。

・Redmine
　オープンソースのプロジェクト管理ソフトの一つ。

・Trello
　カードを動かしながらタスクを視覚的に管理できるカンバン方式のツール。

■ バージョン管理
　ファイル対して、誰が、いつ、なにを変更したか、などの情報を記録することで、過去のある時点の状態を復元したり変更内容の
　差分を表示できるシステム。

集中型
　ファイルのバージョン管理を専用のサーバ（リポジトリ）で一元管理する方式。
　サーバの繋がっていない状態ではファイルの変更情報を記録できない。

分散型
　ユーザがそれぞれPC内にリポジトリのコピーを持つ方式。
　ファイルが更新されるとそれぞれのPC内のローカルリポジトリでファイルの更新を記録し、ある程度作業が進んだら、
　サーバ上のリモートリポジトリに変更内容を反映させます。

Github
　バージョン管理システムで管理するデータをインターネット上で共有、公開することができるネットサービス。

Backlog
　SaaS型のプロジェクト管理ツール

SourceTree
　プログラムのソースコードを管理するGitを簡単に扱うためのツール。

Tortoise SVN
　集中管理型のバージョン管理システム「Subversion」のクライアント

「Git Flow」
　チームでGitを使う際に役に立つ、運用ツールをまとめたツール（プラグイン）

・master
　処理の基本となるデータを集めたテーブルなど、同種のものが複数ある場合に優先されるもの、元になるもの。

・release
　ソフトウェアの公開や販売のこと。

・develop
　新しいものを作り出す事。

・feature
　機能、特性

■ FW／ライブラリ
「FW」
　ファームウェアとは、コンピュータに内臓されているソフトウェアの一種。
　本体内部の回路や装置などの基本的な制御を司る機能。

・Spring Framework
　Javaの開発環境で使用できるオープンソースのフレームワーク

・Struts
　Webアプリケーションフレームワークの一つ。

・JSF（Java Server Faces）
　Java言語を用いてWebアプリケーションを開発するためのフレームワークの一つ。
「ライブラリ」
　ある特定の機能を持ったコンピュータプログラムを他のプログラムから呼び出して利用できるように部品化し、
　そのようなプログラム部品を複数集めて一つのファイルに収納したもの

・Bootstrap
　コンピュータを起動すること。
　OSが操作可能な状態になるまでに自動的に行われる一連の流れ。

・Flyway
　データベースの状態をバージョン管理できる。

・Jackson
　データ構造を見やすく表現できるデータ形式の一つ。

・jQuery
　javascriptの書き方などを簡単にすることができる、javascriptで作成されたライブラリ

・Junit
　javaで書かれたプログラムのテストを自動化するためのツール。

・Log4j
 作成したプログラムが正しく動作しているかを検証する方法。

・Lombok
　java言語におけるボイラープレートコードをソースコードから排除するために使用するライブラリ。
　ボイラープレートコードとは、言語仕様上省くことができない定型的なコード。

・ORM
　データベースに対して、SQLを記述せずにPHPのコードでCRUDを実現する仕組みのこと。
