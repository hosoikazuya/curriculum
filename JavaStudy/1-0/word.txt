
■ IT
　IT　：インターネットとコンピュータを駆使した技術の総称
　ICT　：情報技術を共有するためのコミュニケーションが強調された用語
　IoT　：身の周りのあらゆる物がインターネットにつながる仕組みの事

■ OS
　OS　：ソフトウェアとハードウェアを仲介しコンピュータを動かす重要な役割があるオペレーティングシステムの略
　Windows　：Microsoft社のOS
　macOS (Mac OS X)　：AppleのOS
　UNIX :最古に近いOS
　Linux　：オープンソースのOS

■ 言語・環境
・マークアップ／フロントエンド　：WebサービスやWebアプリケーションで直接ユーザーに目に触れる部分のこと
　HTML　：HyperText Markup Language の略。ウェブページを作成するために開発された言語。
　CSS　：Cascading Style Sheets の略。ウェブページのスタイルを指定するための言語。
　JavaScript　：ウェブページをダイナミックに動かすための言語。
　　jQuery　：JavaScriptの書き方など簡単にすることができるJavaScriptで作成されたライブラリのこと。
　Java SE　：Javaの基本機能をまとめたもの
　
・サーバーサイド　：サーバー上で動くもの
　.NET（C#, VB）　：主にMicrosoft .NET Frameworkというアプリケーション、または開発環境のこと。
　Java　：汎用的でOSに依存しない言語
　　Java SE　：Javaの基本機能をまとめたもの
　　Java EE（Jakarta EE）　：Webアプリケーションの開発に必要な機能をセットにしたもの。
　Perl :CGIを実現するためのプログラミング言語の一つ。
　PHP　：動的にWebページを生成できる言語。
　Ruby　：動的にWebアプリケーションを生成できる言語

■ サーバーソフト　
　Apache　：全世界的に普及しているWebサーバのソフト
　nginx　：シェア第2位のWebサーバのソフト
　IIS　：WindowsのWebサーバソフト
■ SQL
・SQL　：データベース言語の一つ。国際標準化機構（ISO）で規格が標準化している。
　DB2　：米IBM社のリレーショナルデータベース管理システム。
　MySQL　：オープンソースで世界で最も使われているデータベース管理システム。
　　Maria DB　：MySQLから派生したリレーショナルデータベース管理システム。
　Oracle DB　：oracle社が開発したリレーショナルデータベース管理システム。
　PostgreSQL :オープンソースのリレーショナルデータベース管理システム。
　SQLite　：オープンソースで軽量のデータベース管理システム。アプリケーションに組み込まれる。
　SQL Server　：Microsoft社が開発したデータベース管理システム。

・CRUD ：生成。読み取り。更新。削除。データを操作する際に必要な最低限の機能。

・トランザクション　：複数の処理を一つの処理にまとめたもの。
　コミット　：行った処理を確定させること。
　ロールバック　：実行中の処理を取り消すこと。

・排他制御（排他ロック）　：同時アクセスにより不整合が発生しないように他のトランザクションからはアクセスできないように処理され制御すること。
　楽観ロック　：データそのものに対してロックを行わず、更新対象のデータがデータ取得時と同じ状態であることを確認してから更新して整合性を保証する方式。
　悲観ロック　：データにロックをかけて、他のトランザクションから更新されないようにする方式。
・実行計画　：SQLを実行するための計画。
・チューニング　：調整を行うこと。
・パフォーマンス　：性能のこと。
・スロークエリ　：データベース管理システムに対する問い合わせが時間がかかっているもの。
・クエリログ　：履歴のこと

・IDE（統合開発環境）　：ソフトウェア開発に必要なソフトウェアを一つに組み合わせたソフトウェアパッケージ。
　Eclipse　：オープンソースで開発されているソフトウェアIDE。
　NetBeans　：Oracleを中心に開発されているオープンソースの統合開発環境。
　SpringToolSuite　：Eclipseベースの無償の統合開発環境。
・DBツール
　A5:SQL Mk-2　：データベース開発の生産性を向上させるべく開発されたSQL開発ツール。
　pgAdmin　：ネットワークを通じて遠隔のデータベースサーバをGUIで管理できるもの。
　DBeaver　：無料のマルチプラットフォームデータベース。
　MySQL Workbench :統合ビジュアルツール。
　Sequel Pro　：mac用のMySQLのデータをGUI操作で管理するアプリケーション
　TeamSQL　：クライアントツール
■ 肩書
PM　：システム開発プロジェクトの責任者
PL　：プロジェクトチームを管理・統括する
PG（PE）　：SEが設計した設計書に従ってプログラミングを行う仕事。
SE　：システムの設計や開発・テストを行う職種。
デザイナー　：デザインを考案する仕事。
エンジニア　：情報技術者
　フロントエンド・エンジニア　：ユーザーが画面越しに触れる部分の設計や構築を行う職種
　サーバーサイド・エンジニア　：サーバー側で動作するプログラムの開発やデータ処理を行う職種
　バックエンド・エンジニア　：ユーザーからは見えない裏側のシステム部分を構築する仕事。
■ UI（ユーザーインターフェース）
UI　：システムなどとユーザーの間で情報をやり取りする仕組み。
CUI　：キーボード入力されるコマンドによって操作を行うUI。
GUI　：コンピュータの画面上に表示されるボタンなどをマウスで操作するUI。
■ ファイル／ディレクトリ
ファイル　：コンピュータにおけるデータの管理単位の一つ。
拡張子　：ファイルの種類を表す。
ディレクトリ　：複数のファイルをひとまとめにして保管できる保存領域のこと。
ルート・ディレクトリ　：一番上のディレクトリ。
カレント・ディレクトリ　：現在見ているディレクトリのこと。

パス　：ファイルの場所を表す情報　通り道
絶対パス　：URLでページを指定して、目的地がどこにあるのかを確実に伝える情報。住所。
相対パス　：今の場所を基準に目的地が何処にあるのかを伝える情報。案内経路。

■ ネットワーク
LAN　：一定の限定されたエリアで接続できるネットワークのこと。
WAN　：LANとLANを繋げたワイドなネットワークのこと。
クライアント :他のコンピュータから機能や情報の提供を受けるコンピュータのこと。
サーバー　：ネットワーク上のコンピュータの中で、他のコンピュータから指示を受けて、情報を返す役割を持つコンピュータのこと。

■ 各種プロトコル
HTTP　：データの送受信を行うためのプロトコル。
FTP　：ファイル転送プロトコル。
HTTPS　：HTTPが暗号化されているプロトコル。
SSL　：情報を暗号化し送受信できるプロトコル。
SSH　：ネットワークを経由して遠隔操作ができるプロトコル。

ポート、ポート番号　：ポート（接続口）に割り振られている識別用の番号

HTTP、FTP、SSHの各ポート番号　：HTTP（TCP80）　FTP（TCP20.21）　SSH（TCP22）

ファイアウォール　：外部ネットワークからの攻撃から内部ネットワークを保護するシステム。
TCP　：インターネットプロトコルセットの中核をなすトランスポート層のプロトコル。

■ IPアドレス
IPアドレス　：インターネット上に接続された機器に割り当てられた固有番号
グローバルIPアドレス　：インターネット上で重複しないIPアドレス。
プライベートIPアドレス　：LAN内で自由に使用できるIPアドレス。
■ ドメイン
ドメイン　：インターネット上の住所
DNS　：インターネットなどのネットワーク上でドメイン名とIPアドレスの対応関係を管理するシステム。
VPN　：インターネットなどを仮想的に専門線のようなプライベートネットワークとして利用するサービス。
ブリッジ　：２つのネットワークを接続し、一つの大きなネットワークにすること。
プロキシサーバー　：インターネットに接続できないコンピュータの代わりに接続しWebサイトへのアクセスを行うサーバーのこと。
■ 電子メール
SMTP ：メールを送信するための通信プロトコル。
POP　：メールを受信するための通信プロトコル。
IMAP　：電子メールを受信するためのプロトコル。メール受信サーバにアクセスし、必要分を選んで手元に受信する手順。
■ セッションとクッキー
セッション　：セクションIDを使用して情報交換をする一連の流れ
クッキー　：HTTPを拡張してアプリケーションとブラウザの間で情報交換できるようにしたもの。
キャッシュ　：データを一時的に保存して、データ処理速度を早める仕組み。
■ DB（データベース）について
インデックス　：個々の要素を区別するために付けられた通し番号などの識別番号。
■ テストについて
フロー　：コンピュータプログラムを構成する命令や処理の流れのこと。
■ チケット管理ツール
タスク　：コンピュータが処理するプログラムの処理の単位のこと。
■ バージョン管理
Git　：オープンソースの分散バージョン管理システムの一つ。
SVN　：ファイルの変更履歴を管理してくれるシステムの一つ。
デグレ　：修正や変更によって正常に作動していた機能が動かなくなること。
マージツール　：FDLファイルまたはDTDファイルの変更内容をMDLファイルにマージできる。
